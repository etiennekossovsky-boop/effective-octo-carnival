# effective-octo-carnival
Emojicom  Emojicom est un langage visuel universel, ouvert et collaboratif.  Il organise les emojis en séquences cohérentes (“serials”) capables d’exprimer simplement des idées complexes, au-delà des langues, des cultures et des alphabets.  Emojicom n’est pas une collection d’emojis.  C’est une grammaire, une méthode, et un bien commun.

🌍 Vision

Créer un langage visuel partagé, accessible à toutes et tous, permettant :

- de transmettre des idées complexes avec des séquences simples ;
- de faciliter la communication interculturelle ;
- d’offrir un outil universel pour l’éducation, la médiation, l’inclusion et la créativité ;
- de constituer un corpus ouvert, vivant, enrichi par la communauté mondiale.

---

🧩 Principes de base

1. Universalité — Les emojis sont déjà un alphabet global. Emojicom les organise. 
2. Simplicité — Chaque séquence suit une structure claire et stable. 
3. Lisibilité — Une même idée = une même séquence. 
4. Évolutivité — Le langage grandit par contributions ouvertes. 
5. Neutralité culturelle — Les séquences visent la compréhension globale. 
6. Open Source — Le projet appartient à toutes et tous.

---

📐 Structure d’une séquence (serial)

Une séquence Emojicom suit un format standardisé :

`
[Contexte] → [Action] → [Valeur] → [Résultat]
`

Exemples :

- 🤝 ➡️ 🌍 = coopération globale 
- ⚠️ 💡 🛠️ 🙂 = problème → idée → solution → apaisement 
- 👥 🔄 🛠️ = responsabilité partagée 

La grammaire complète est dans /core/grammar.md.

🤝 Contribuer

Emojicom vit grâce à la communauté. 
Toute contribution est la bienvenue : séquences, corrections, idées, discussions, traductions.

Avant de proposer une nouvelle séquence, merci de :

2. Vérifier qu’une séquence similaire n’existe pas déjà 
3. Utiliser le template officiel (/proposals/template.md) 
4. Ouvrir une issue ou une pull request

Les règles de contribution complètes sont dans la Charte Emojicom (/docs/howtocontribute.md).

---

1. Respect de la grammaire

Toute séquence doit respecter les règles définies dans /core/grammar.md.

- Structure stable (ex. : Contexte → Action → Valeur → Résultat)
- Emojis choisis pour leur universalité
- Pas de références culturelles trop spécifiques
- Pas d’ambiguïté volontaire

---

2. Neutralité culturelle

Emojicom vise la compréhension globale.

- éviter les symboles religieux, nationaux ou politiques 
- privilégier les concepts universels (coopération, aide, danger, joie…) 
- éviter les emojis dont le sens varie fortement selon les cultures 

---

3. Clarté et justification

Toute proposition doit inclure :

- la séquence proposée 
- sa signification 
- une justification claire 
- les variantes possibles 
- les limites ou ambiguïtés éventuelles 

Utiliser le modèle /proposals/template.md.

---

4. Discussion ouverte

Les propositions sont discutées publiquement via les issues.

- bienveillance 
- argumentation constructive 
- écoute des autres interprétations 
- recherche du consensus 

---

5. Validation

Une séquence est acceptée lorsqu’elle :

- respecte la grammaire 
- est compréhensible globalement 
- a été discutée et améliorée 
- a reçu l’accord du comité de validation 

Elle est alors intégrée dans /serials/vX.Y/.

# Emojicom 🌌🌍🌝🌐

Emojicom est un langage visuel universel, construit collectivement, pour exprimer des idées complexes avec des séquences simples d’emojis appelées **serials**.

> Un pont, pas un remplacement. Un outil de plus pour se comprendre.

## 🧱 Grammaire v0.1

Format standard : `[Contexte] → [Action] → [Valeur] → [Résultat]`

- **Contexte** : la situation de départ, le cadre
- **Action** : ce qui se passe, ce qui change  
- **Valeur** : l’intention, le principe, la qualité mise en jeu
- **Résultat** : l’effet final, l’état d’arrivée

Détails : `/core/grammar.md`  
Syntaxe : `/core/syntax.md`  
Principes : `/core/principles.md`

# Emojicom Syntax v0.1
Copyright 2026 Emojicom
Licensed under the Apache License, Version 2.0

## 1. Objectif
Ce document formalise les conventions de syntaxe pour Emojicom v0.1. 
Principe : stabilité et simplicité avant expressivité.

## 2. Flèches et transitions
Uniquement 2 symboles autorisés en v0.1 :

| Symbole | Sens | Usage |
| --- | --- | --- |
| ➡️ | Passage, transformation, séquence | Marque un passage clair d’un état à un autre |
| 🔄 | Échange, réciprocité, circularité | Marque une relation mutuelle ou cyclique |

Exemple : 🤝 ➡️ 🌍 = coopération qui se déploie vers le monde

## 3. Répétition et intensité
La répétition d’un emoji marque l’intensité, la pluralité ou le caractère massif.

Exemple : 🔥🔥 = crise majeure, urgence forte

Règle v0.1 : Répétition interdite dans les serials officiels sauf si documentée et justifiée dans la proposition. Les forks expérimentaux peuvent explorer.

## 4. Modificateurs v0.1
4 modificateurs autorisés, usage parcimonieux :

| Modificateur | Sens | Position |
| --- | --- | --- |
| ❗ | Alerte, urgence, attention | Avant ou après le serial |
| ❓ | Questionnement, incertitude | En fin de serial |
| ♻️ | Durabilité, circularité, recyclage | Qualifie une Valeur ou Résultat |
| 🔍 | Analyse, enquête, observation | Qualifie une Action |

Exemple : ⚠️ 🔍 🛠️ 🙂 = problème analysé → action → apaisement

## 5. Règles générales
1. 2 à 4 emojis par serial en v0.1. Pas plus.
2. Pas d’emoji de ponctuation autre que ceux listés ci-dessus.
3. L’ordre `[Contexte] → [Action] → [Valeur] → [Résultat]` prime sur tout.
4. Tout ajout à la syntaxe passe par une issue + discussion + accord.

## 6. Évolutions futures
v0.2+ pourra introduire : temps, nuances, compositions, emojis combinés.
v0.1 reste volontairement minimale pour garantir l’adoption.

# Cooperation / Coopération
Copyright 2026 Emojicom
Licensed under the Apache License, Version 2.0

## Séquence exacte
🤝 ➡️ 🌍

## Définition courte
Coopération globale. Déploiement d’un lien de collaboration à l’échelle du monde.

## Explication détaillée
- **Position 1 🤝 Contexte** : Coopération, lien, accord entre parties. Point de départ = relation.
- **Position 2 ➡️ Action** : Déploiement, extension, passage à l’échelle. Mouvement du local vers le global.
- **Position 3 🌍 Valeur** : Monde, globalité, ensemble des vivants/humains. Qualité mobilisée = échelle planétaire.
- **Position 4 Résultat** : Implicite. Le résultat est la coopération mondiale elle-même. L’état d’arrivée est contenu dans la séquence.

## Cas d’usage typiques
1. Décrire un projet open source international
2. Exprimer un appel à l’entraide planétaire
3. Qualifier une gouvernance partagée
4. Tagger du contenu lié à la collaboration globale

## Limites ou ambiguïtés
1. Ne précise pas le type de coopération : économique, sociale, écologique.
2. Ne distingue pas coopération choisie vs subie. 
3. Pour préciser, composer avec un autre serial ou attendre v0.2.

## Statut
Officiel v0.1. Serial fondamental.

# Ecology / Écologie
Copyright 2026 Emojicom
Licensed under the Apache License, Version 2.0

## Séquence exacte
🌍 🌱 ❤️

## Définition courte
Soin du vivant. Posture de respect et de régénération envers la Terre et ses écosystèmes.

## Explication détaillée
- **Position 1 🌍 Contexte** : Monde, Terre, cadre planétaire. On parle du système global.
- **Position 2 🌱 Action/Valeur** : Vivant, croissance, régénération, durabilité. Fait office d’action ET de valeur en v0.1.
- **Position 3 ❤️ Valeur/Résultat** : Soin, bienveillance, amour. L’intention et l’état visé.
- **Position 4 Résultat** : Implicite, fusionné avec ❤️. L’apaisement et l’harmonie sont contenus dans le soin.

Type : Serial de Valeurs. Décrit une posture plus qu’un processus.

## Cas d’usage typiques
1. Manifeste d’un projet écologique
2. Qualifier une action régénérative
3. Exprimer une éthique du vivant
4. Tagger du contenu lié au soin de la Terre

## Limites ou ambiguïtés
1. Ne distingue pas écologie de conservation vs régénération.
2. Ne précise pas l’échelle : locale ou globale.
3. Pour un processus, utiliser ⚠️ 🌱 🛠️ 🙂 = problème écologique → régénération → action → apaisement.

## Statut
Officiel v0.1. Serial fondamental.
